##########################################################
##	##	##	WORKAROUNDS	##	##	##
##########################################################
#  - Kills any wlrun.exe & AnalysisUI.exe running processes processes at the beginning of each build
#  - writes LRR in a #TEMP# directory, since Analysis can't take any argument when launched in command line
#  - Analysis MUST be parametered with a default template creating LRA & HTML reports in the same #TEMP# directory as set in the Plugin configuration
#  - Using jfreesvg-2.1.jar lib to display the graphs. Installed in a project-built-in maven repository :
#	# mvn install:install-file -Dfile="[/path/to/jfreesvg-2.1.jar]" -DlocalRepositoryPath=lib -DgroupId=org.jfree -DartifactId=jfreesvg -Dversion=2.1 -Dpackaging=jar -Dcreatechecksum=true
	# include this section in your POM.xml :
	#    <dependency>
	#      <groupId>org.jfree</groupId>  
	#       <artifactId>jfreesvg</artifactId>  
	#       <version>2.1</version> 
	#       <scope>runtime</scope>
	#   </dependency> 
#  - No way to use ProminentProjectAction (no Action was brought up to the job level) nor TransientProjectActionFactory (Plugin was displayed in every JENKINS jobs)
#


##########################################################
##	##	##	LIMITATIONS	##	##	##
##########################################################
#  - It's only possible to use the Analysis Defaut Template to generate LRA & HTML reports
#  - Configure Analysis Template to save LRA in a TMP directory (same as used by WLRUN to generate LRR
#  - CANNOT BE DEPLOYED ON A MASTER/SLAVE CONFIGURATION (STANDALONE ONLY)
#  - Since any Wlrun.exe running process is killed at the launch of a build, Jenkins may be configured with one executor only
#  - Results extraction are based on the parsing of the path/to/LRA/LRA.asc file, assuming each Transaction-related metric in the file adopts the following format : RowNo[0-9]+=Transaction Name#Minimum#Average#Maximum#Std. Deviation#90 Percent#Pass#Fail#Stop
